Client

new Client(transportSocket, options)

STOMP client
Parameters:
Name Type Argument Description
transportSocket stream.Duplex TCP socket or another type of Stream.Duplex object that's ready to read and write. It's acceptable to pass a socket object that's in connecting state.
options Object <optional>
Properties
Name Type Argument Default Description
heartbeat Array <optional>
 [0,0] Set the preferred heart beat timings, in milliseconds. The first element is the send frequency. The class will try to ensure the send frequency is reached by sending frame trailer bytes if the application is not creating busy enough traffic. The second element is the expected receive frequency; i.e. the client expects to see an increase in the number of bytes received on the socket every n milliseconds). To understand how the client and server negotiate timings read STOMP 1.1 spec on Heart-beating
heartbeatDelayMargin Number <optional>
 100 Milliseconds added to the the heart-beat receive frequency to account for network delay.
